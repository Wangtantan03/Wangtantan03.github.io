# 浏览器的工作流程

PC端的浏览器输入url后，按下回车到整个页面渲染出来的这个过程中都发生的什么？

明确几个概念

域名（domain）：一台或者一组服务器的名称，用来确定服务器在internet上的位置

url：统一资源定位符，用来确定某一文件的具体位置

IP和域名是否是对应关系：否，即可以把多个提供相同服务的服务器IP设置为同一个域名，同样，一个IP可以绑定多个域名，数量不限。

什么是DSN：域名解析服务，即将主机名转化为IP地址。 例如：将主机名http://www.cnblogs.com/转换为IP地址：211.137.51.78

  1. 在浏览器地址栏中输入url，解析url，检测url地址是否合法；
  2. 浏览器查找域名的IP地址；
  • 依次查看浏览器缓存、操作系统缓存、路由器缓存和网络运营商缓存（ISP）有相应的域名、IP对应关系。若有，则向其IP地址发送http请求；
  • 若无，则将域名（domain）发送给DNS（域名服务器）进行解析，将域名解析成对应的服务器IP地址，发回浏览器。（递归查询如下图所示：）
  1. 浏览器拿到IP后，向web服务器发送tcp连接请求，建立tcp三次握手；
  2. 握手成功后，浏览器向服务器发送http请求，请求数据包；
  3. 服务器处理收到的请求，发回http响应；
  4. 浏览器收到http响应；
  5. 浏览器解码响应，若响应可缓存，则存入缓存；
  6. 浏览器发送获取HTML中的对象的请求（如：html、css、JavaScript、图片等）；
  7. 浏览器发送进一步的异步（AJAX）请求；
  8. 至此页面渲染结束。；
