# selenium实战篇

# selenium web登录测试

今天对公司的运营管理系统进行回归测试时候，涉及到登录的测试，于是想利用空余时间使用 python+selenium 自己动手实现自动化测试。下面对自己的 实现过程以及思路做一下总结，方便日后回顾。

# 1. 创建一个简单的登录测试

实现简单的登录测试

学习时用了一个开源的管理系统的demo，通过python+selenium实现登录测试，包括输入账号，输入密码，登录以及注销的功能测试

   from selenium import webdriver
   import time

   driver = webdriver.Chrome(executable_path="/Users/wangtantan/Desktop/Selenium/Drivers/chromedriver")
   driver.implicitly_wait(10)     # 隐式等待，10秒钟内找到了元素就开始执行，否则就超时
   driver.maximize_window()

   driver.get("https://opensource-demo.orangehrmlive.com/")      # 这里用了一个开源的管理系统的demo
   driver.find_element_by_id("txtUsername").send_keys("Admin")
   driver.find_element_by_id("txtPassword").send_keys("admin123")
   driver.find_element_by_id("btnLogin").click()
   driver.find_element_by_id("welcome").click()
   driver.find_element_by_link_text("Logout").click()

   # 完成上面的动作之后，为了更加明显显示，延长一些显示时间
   time.sleep(2)

   driver.close()  # 关闭驱动程序
   driver.quit()   # 退出驱动程序
   print("测试完成")

# 知识点总结

# unittest中的setUp()和tearDown()函数

调用次数：

在每个测试方法（用例）运行时被调用一次

  • setUp()函数的定义：

该函数是在众多函数里最先被调用的函数或者说是在一个类里最先被调用的函数，每执行完一个函数都要去调用一下setUp()函数，然后再执行下一个函数，程序中有几个函数就要调用该函数几次， 与函数放置的位置无关。

  • setUp()函数的使用场景：

例如在自动化测试常常使用setUp（）函数进行测试前的初始化工作，例如：在接口测试之前进行一些前置的参数赋值，数据库操作等。

  • tearDown()函数的定义：

该函数在众多函数执行完之后被执行，在一个类中无论有多少个函数，也不论该函数放在哪个位置，总是最后一个被执行。

  • tearDown()函数的使用场景：

主要是在测试之后进行一些清除工作，比如浏览器关闭退出，参数还原或销毁，数据库的还原恢复等。
