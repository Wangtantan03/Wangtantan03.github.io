# 冒泡排序法

# 基本思想

  1. 比较相邻的元素。如果第一个比第二个大，就交换他们两个。
  2. 对每一对相邻元素做同样的工作，从开始第一对到结尾的最后一对。在这一点，最后的元素应该会是最大的数。
  3. 针对所有的元素重复以上的步骤，除了最后一个。
  4. 持续每次对越来越少的元素重复上面的步骤，直到没有任何一对数字需要比较。

# 算法面试题

利用python并使用冒泡排序法对list中的元素排序，同时编写unittest单元测试用例进行验证

# -*- coding:utf-8 -*-
import unittest

def bubble_sort(arr):
    length = len(arr)
    for i in range(length - 1):  # 决定比几次（行）
        for j in range(length - i - 1):  # 怎么比
            if arr[j] > arr[j + 1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
    print(arr)


class SortTestCase(unittest.TestCase):

    def test_bubble_sort(self):
        test_data_1 = [6, 5, 3, 7, 8, 1, 24]
        copy_of_data_1 = test_data_1[:]  # 拷贝test_data_1

        bubble_sort(test_data_1) # 使用冒泡排序法进行排序
        copy_of_data_1.sort()  # 当sort函数默认情况下，对原列表进行升序排序
        self.assertEqual(test_data_1, copy_of_data_1)

if __name__ == '__main__':
    unittest.main()

# 时间复杂度

  • 最优时间复杂度：O(n) ，因为最好的情况是要排序的数据已经有序了,只需要进行一次冒泡操作,故时间复杂度是O(n)
  • 最坏时间复杂度：O(n^2)，因为最坏的情况是要排序的数据是倒序的,要进行n次冒泡操作,故时间复杂度是O(n^2)
  • 平均时间复杂度：O(n^2)
  • 稳定性：稳定的排序

# 空间复杂度

  • 算法内存的消耗可以用空间复杂度来衡量
  • 冒泡排序过程只涉及相邻数据的交换操作,只需要常量级的临时空间,所以该算法的空间复杂度为O(1),是一个原地排序算法.

# 插入排序法

# 基本思想

  1. 抽象概念：插入排序（Insertion Sort）的算法描述是一种简单直观的排序算法，它的工作原理是通过构建有序序列，对于未排序数据，在已排序序列中从后向前扫描，找到相应位置并插入。
  2. 类比打扑克牌：右边新摸来的牌，与左边有序的老牌依次进行比较，最终插到合适的位置。
  3. 实现过程：
  • 假设队列左边的元素是已经排序过的元素
  • 依次遍历已排序过的元素右边的元素，将该元素与左边已排序的元素做比较，直到找到比它小或等于它的元素插入在其后面
  • 重复第二步，直到所有的元素全部排序完成

# 算法面试题

利用python对list中的元素进行插入排序

# -*- coding:utf-8 -*-

def insertion_sort(arr):

    # 从要排序的列表第二个元素开始比较
    length = len(arr)
    for i in range(1, length):
        j = i
        # 从大到小比较，直到比较到第一个元素
        while j > 0:
            if arr[j] < arr[j-1]:
                arr[j-1], arr[j] = arr[j], arr[j-1]
            j -= 1
    return arr

arr = [6, 5, 3, 1, 8, 7, 2, 4]
print(insertion_sort(arr))

  • 平均时间复杂度：O(n^2)

# 二分查找算法

# 基本思想

  1. 概念：二分查找又称为折半查找（Binary Search）是一种效率比较高的查找方法，它要求线性表必须采用顺序存储结果，而且表中元素按关键字有序排列。
  2. 优点：该算法的比较次数较少，且查找速度快。
  3. 缺点：要求待查寻的表必须为顺序表，多适用于不经常变动而查找频繁的有序列表。
  4. 实现过程：
  • 假设表中的元素按升序排列，将表中间位置记录的关键字与待查找的关键字进行比较
  • 如果中间位置记录的关键字与待查找的关键字相等，则查找成功
  • 否则利用中间位置记录的关键字将表分为前、后两个子表
  • 如果中间位置记录的关键字大于查找的关键字，则进一步查找前一子表，否则查找后一子表
  • 重复上面的过程，直到找到待查的关键字，此时查找成功；或者直到子表不存在为止，此时查找不成功

# -*- coding:utf-8 -*-
# 二分查找----递归版本
# 函数传递的参数为list和待查找的具体值value
# 首先将有序序列对半砍，然后判断中间位置的值是否与待查找value相等，大于右边查，小于就在左边查
# 递归调用函数，参数变为新的list

def binary_search(alist, value):
    '''二分查找'''

    n = len(alist)  # 注意：len()函数返回的是对象的长度，若对象为字符串则返回字符串长度，若对象为列表，则返回列表元素个数
    mid = (n - 1) // 2  # //：表示整除，返回不大于结果的一个最大整数 /： 表示浮点数除法，返回浮点结果
    while n > 0:  # 递归的终止条件 即大于等于一个元素的时候
        if alist[mid] == value:
            return True # 查找成功返回Ture
        elif value < alist[mid]:
            # -----中间值小于value，在左半部分查找-------
            return binary_search(alist[:mid], value)
        else:
            # -----中间值大于value，在右半部分查找-------
            return binary_search(alist[mid + 1:], value)
    return False # 查找失败返回False
testlist = [2, 3, 5, 10, 15, 16, 18, 22, 26, 30, 32, 35, 41, 42, 43, 55, 56, 66, 67, 69, 72, 76, 82, 83, 88]
